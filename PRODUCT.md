# PSF Scan

## Users

光学/显微学单人实验台用户。每天数十次重复同一动作：让位移台带样品穿过焦点，相机连续抓帧，从图像里的光斑形态判断离焦量与对中。常在台面荧光灯、显微镜旁的局部照明、普通办公室屏幕亮度之间切换。一手握 stage 手柄，一只眼看目镜或屏幕，对设备熟，对花哨界面零容忍。

## Product Purpose

把"移动 → 抓帧 → 判读"这套循环压缩到最少的视觉与认知负担。核心只有三件事必须立刻可见：当前 PSF 强度图、当前位置、扫描进度。其它都让位。

## Register

Product。设计服务于实验流程，不抢走对图像和数值的注意力。

## Tone

科研绘图工作台。冷静、准确、接近 Matplotlib / ImageJ / LabVIEW 这类实验软件的可读性，界面像可靠的仪器控制面板，但图像和图表必须像论文前的数据检查图一样清楚。控件可以紧凑，文字必须清楚，数值必须稳定。

## Anti-references

- 黑底科研软件、终端绿/青配黑底：图像强度由 colormap 表达，不靠 UI 背景装气氛。
- 魁北克蓝 + 黄金（"专业 SaaS" 的科学版）。
- 大而圆润的阴影卡片、icon-标题-描述三件套。
- 任何会让位移台校准的人觉得"这个软件比我聪明"的语气。
- 装饰性渐变、光晕、背景模糊、动画过渡 > 200ms。

## Strategic Principles

1. **PSF 图最大、控件让位**：相机视图占主屏 > 60%，所有控件加起来占剩下不到 40%。
2. **绘图用浅色画布**：实时图像、PSF 图、XY / Z 轨迹默认在浅色中性画布上显示；轴线、网格、标尺低对比但可读。不要把图表区做成黑底。
3. **数值用等宽**：所有显示位置/曝光/进度的数字必须等宽，校准时眼睛要在三位小数间扫过来扫过去，宽度抖动是噪音。
4. **状态就是颜色**：当前位置 = 温和蓝色；已扫描点 = 低饱和绿色；未访问点 = 中性灰。三色互斥，全屏永远只有一个意义同时活着。
5. **零层叠**：不要嵌套 box / 嵌套卡片。section 只用一条横线划开，足够。
6. **亮而不刺眼**：背景使用带暖灰倾向的浅色，避免纯白大面积照眼。文字保持高对比，强调色克制使用。
